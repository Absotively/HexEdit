Number Find Page

 

 

This page is used to search for numeric values.

 

Find

Enter the number (in decimal) to look for here.  The format of the number depends on the Format selected (see below).  A floating point format (IEEE or IBM) can specify a sign, a fractional part (following a decimal point) and an signed exponent (following an E or e).  A signed integer can specify an initial minus sign.

 

Note that for floating point formats the number you think you need to search for may not have the exact bit-pattern that you expect (see How Do I Search for a Floating Point Number?).

Find Next

Searches for (and selects if found) the next occurrence of the number.

Help

Invokes help for this page.

 

Format

This is the format of the number that you are searching for.  Note that all formats can come in various sizes (see Size below).  A signed integer is in 2's-complement format.  See Numbers for more information on number formats.

Size

The format, size and endianness are used to exactly specify the format and number of bytes to search for.

Big-endian

If this option is specified then the number is in big-endian format.  Note that IBM floating point numbers are usually in big-endian format, but integers and IEEE floats on Intel machines are usually in little-endian format.  See the section on Byte Order in Numbers for more information.

 

Direction Up/Down

These radio buttons say whether the search moves forward or backward in the file.

Scope: To Mark

Sometimes it is useful to restrict a search to just part of a file, particularly if the file is very large.  This option means that the search is only done in those bytes from the current cursor position to the mark in the active file.  You can see how many bytes will be searched by viewing the "Distance to Mark" panes of the status bar.

 

This option is disabled if the current cursor position is on the wrong side of the mark (depending on the search direction - see above)

Scope: Whole file

Use this option if you want to search the whole file irrespective of the current cursor position in the file.  Depending on the direction of the search (see above) the file is searched starting at the first or last byte.  After the first successful search with this scope is complete the scope then changes to "To Start of File" or "To End of FIle" (depending on the current search direction) so that you can just hit the "Find Next" button to find the next occurrence.

Scope: To Start/End of File

The scope of the search is from the current cursor position to the end of file (or start of file if the current search direction is upwards).  When the end (or start) of file is reached you are asked if you want to continue the search from the other end of  the file back to the start.

Scope: All Open Files

When this scope is used the current file is searched from the current cursor position to the end (or start if direction is upwards), and if nothing is found then other open files are searched in turn.  Finally, if no occurrences are found in all other files then the start (or end) of the current file is searched back to the initial position.

Alignment

The alignment radio buttons can further restrict a match depending on the alignment of the matched number.  A word alignment means that the number must start on a word boundary (ie, at an even address).  Similarly double word and quad word restrict matches to 4 and 8 byte boundaries.  The default is byte alignment which uses no alignment restrictions.
