Selecting and the Clipboard

 

To select part of a file you can use the mouse or the keyboard. To use the mouse, click the first byte you want to select and drag the mouse to the last byte. Alternatively, you can click (and release) on the first byte then shift-click (hold down the Shift key while you click) the last byte. Shift-clicking is very useful if you need to select a large area since you can use the scroll bar to quickly move to the other end of the desired range.

 

To use the keyboard you can use the Shift key with the normal cursor movement keys. For example, to select everything from the current cursor position to the end of the file press Shift+Ctrl+End.

 

Once selected you can cut or copy the bytes to the clipboard in the normal manner. Note that you cannot cut if the window is in either of the read-only or overtype modes since this will delete bytes from the file. You can also perform many other operations with selected text such as write it to a file, print, compress, export etc.

 

Selection Length

 

While you are making a selection a small "tip" window is displayed which displays the length of the selection (in hex or decimal depending on how addresses are currently displayed).

 

You can bring this tip window up at any time by holding down the "Shift" key.

 

There is also an option for showing what power of 2 the selection length is divisible by.  This is useful as some commands only work with a selection length which is a multiple of 2, 4 or 8.  This option can be turned off in Workspace Display page of the Options dialog.

 

Highlighter

 

The highlighter has already been mentioned but it can also be viewed as allowing a disconnected or discontiguous selection (ie all the selected bytes do not have to be next to each other).

 

You can use the highlighter to mark particular important bytes within a file.  Clicking the highlighter, highlights the selected bytes.  To quickly create several highlights, turn on highlighter mode (click the highlighter button when there are no bytes selected), thence all subsequent selections made will be highlighted.  Highlighter mode can be used when comparing files to highlight all differences that were found.

 

Clipboard Formats

 

When HexEdit places data on the clipboard it does so in two formats. One format is ASCII (actually ANSI) text which allows other programs (such as word-processors and editors) to read it. But in text format some control characters, such as the character with ASCII value zero, cannot be stored. For this reason HexEdit also stores the data in a binary format, allowing you to cut and paste binary data within HexEdit without losing information. (Note that this binary format is the same as is used in the Microsoft Developer Studio binary file editor.)

 

Note that if you are in EBCDIC mode the text format data placed on the clipboard is converted from EBCDIC to ASCII. (The binary format data is still the actual binary data values.) This allows you to copy EBCDIC text to any Windows editor or word-processor. As for ASCII control characters, any invalid EBCDIC characters are silently ignored. Also, if there is no binary data format on the clipboard (ie, the data on the clipboard was not placed there by HexEdit) and you’re in EBCDIC mode then the text is converted to EBCDIC when pasted.

 

Sometimes you may actually want the hex data as the hex digits so you can paste it into a text file. To do this, copy it to the clipboard as hex text. The text is copied with spaces between the pairs of hex digits and with the same line length as shown on screen. This is also useful if the some bytes of the file contain an address of data elsewhere in the same (or another) file. Copy the address as hex bytes, and paste it into the Hex Address Tool and press Enter to jump to the address.

 

A similar option to copying as hex text is to copy as C/C++ source code. This text is suitable for initialising many different sorts of array in C, C++ and other programming languages.  There are many very flexible options - for example you can create CSV integer or floating point data suitable for importing into other programs such as a spreadsheet.

 

The “Paste As” menu items ensure that you paste text (and not binary data) and it is pasted in the desired character set – ASCII, Unicode or EBCDIC. These commands can be used to convert a large amount of text from one format to another. For example, to convert text from EBCDIC to ASCII you need to cut it to the clipboard while in EBCDIC mode and then use the “Paste As ASCII” command. You cannot simply change to ASCII mode and paste, as the binary format data placed on the clipboard by HexEdit takes precedence, and the data will be pasted without conversion.

 

Working with Large Files

 

The clipboard is great for working with small to medium sized blocks of data, but will not work for blocks of more than about 16 Mbytes.  Furthermore, when you paste data into HexEdit the pasted data is stored in memory.  If you paste a lot of large blocks of data you will cause Windows to slow down or even crash through memory exhaustion.

 

The Append to File and the Append to Same File commands were created to avoid these problems.  With these commands you can easily build multi-Mbyte or even multi-Gbyte files easily, by building a file from large blocks appended to a file in order.

 

Another problem with large files is that the scrollbars are virtually useless.  In a multi-Gbyte file there are many Mbytes between each "notch" of the scroll bar, and it can take hours to page to the address you want.  Of course, you can easily jump to an address with one of the Hex or Decimal Address tools or the Go To command.  But how do you select a large block since you need to use the vertical scroll bar before shift-clicking?  The solution is to jump to one end (using a jump tool or the Go To command) and mark the position (using the Mark command), then jump to the other end and use the "Extend to Mark" command.
