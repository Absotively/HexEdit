Quick Start

 

HexEdit is a binary file editor that looks similar to many other such programs. However, it is generally easier to use and has many useful features not found in many, or even any, other hex editors. This is a brief introduction to some of its more interesting and useful facilities.

 

If you are familiar with binary files and at least one Windows word-processor or editor then you can probably use most of HexEdit’s features straight away, but you may still like to read this section to get the most out of the program. If you don’t even want to read this then just leave the “Tip of the Day” feature enabled until you start seeing the tips repeated.

 

If you are not familiar with running Windows software and/or this type of program you should read the help topics covered in the Overview of HexEdit rather than this concise guide. If you are unfamiliar with binary files then you may need to read the Overview of Binary Files section too.

 

If you use HexEdit only occasionally you will find its extensive on-line help very useful. You can usually find help at any time by pressing F1. Pressing Shift+F1 enters help mode whence you may click on a button or other object you want help on. To get help on a menu item hold the mouse over it and press F1.  Dialogs also provide popup help for all their controls, which is accessed through the F1 key or the “?” button on their title bar, or by right-clicking and selecting "What's This".

 

 

Running HexEdit

 

Apart from the normal ways of starting HexEdit (via icon and Start Menu), you can also run it from the command line. This allows you to open multiple files including using wild-cards. There is an option to prevent multiple instances from running – if this option is on and you try to open HexEdit when a copy is already running then the 2nd copy closes, but any files specified are opened in the already running copy.

 

Another option allows you to add HexEdit to Windows Explorer shortcut menus – when you right click a file one of the options is “Open with HexEdit”. You can also drag and drop files onto the HexEdit main window to open them.  See System Options page.

 

Typically you will open files using the File Open command.  This uses the standard Windows File Open dialog. This dialog has many facilities.  For example, you can view a detailed list of files, sort by size, date of last change etc, and resize the window to see more files.  The filter list that allows you to view different types of files (ie, the drop down list called something like "Files of Type") can be completely customized in HexEdit using the Filters page in the Options dialog - this is very useful if you often deal with the same types of files.  You can also select multiple files to open using the usual Shift+click and Ctrl+click.

 

The file selection dialog also caters to almost any conceivable file management task.  You can create new folders using the Create Folder button.  You can delete, rename, move etc files by right-clicking and selecting from the context menu.  You can even change file attributes using the Properties option.

 

 

Tool Bars

 

There are several built-in toolbars of which only the Standard Toolbar and Edit Bar are displayed by default. The Standard Toolbar has several buttons for changing how the display looks. A useful one is the Autofit button , which allows you to easily adjust the number of columns by dragging the sides of the window.  You can also adjust which areas you want to see (hex, character, or both) using the Area Selection drop-list , or choose the oft-requested stacked mode.  The Character Set drop-list allows you to select the character set (ASCII, IBM/DOS, etc) that is used to display characters.  EBCDIC mode affects how string searches are performed and how text is copied to the clipboard. These and other display options, such as the number of columns, and column grouping are found in the Window Display Options page of the Options Dialog.

 

Another nicety is the ability to change to any color scheme using the drop-down color scheme list on the toolbar.  You can create any number of custom color schemes (or use the predefined ones).  Among other things color schemes allow you to set byte ranges to be displayed in different colors to easily spot oddities and problems.

 

 

The Edit Bar has the Hex and Decimal Address Tools . . These can be used to see the current address or quickly jump to an address (with a drop-down history list).  They also double as a base conversion tool as typing a number (or even an integer expression) in one, automatically updates the other.

 

Another tool on the Edit Bar, the Find Tool , allows you to search for a sequence of bytes. You can enter the bytes in hex or as a string. To do a string search type “=” at the start of the string. For a case-insensitive string search use a tilde (~) instead. When EBCDIC mode is on the string searches use EBCDIC characters. More search options (including Unicode searches) are available with the Find Dialog (see Basic Find page, Hex Find page, etc).

 

The Format Bar has buttons for changing the display and includes tools for changing the font and font size .  The Navigation Bar has buttons for moving about including several bookmark commands and a tool for jumping to any bookmark in the current file .

 

The Customization dialog allows you to create your own toolbars (and menu-bars) and completely customize them. See the Customization section below. Right-clicking on any toolbar allows you to show or hide any of the toolbars, or open the customization dialog.

 

 

Status Bar

 

The left side of the Status Bar is the message area. It displays a short description of menu items and toolbar buttons as you move the mouse over them. The message area also sometimes displays a progress bar while a process is executing or shows a message indicating an unusual situation that is not serious enough to warrant displaying a message dialog.

 

To the right of the message area, the status bar is divided into several panes or indicators. The first indicator displays the number of search occurrences that were found in the current file. This says “OFF” if background searches (see below) are turned off or shows a progress bar if a background search is in progress. Double-click this indicator to start a new search.

 

The next indicator shows information about the current byte at the cursor (the hex, decimal, octal, and binary values and the ASCII or EBCDIC character). To get even more information double-click this indicator and the Properties Dialog will appear. (See Properties section below.)

 

There are two “Distance to Mark” indicators showing the number of bytes between the mark and the cursor, in hex and decimal. The mark can be positioned anywhere in a file using the “Mark Position” command – by default this is associated with double-clicking on the byte.  As usual the hex distance is shown in red and the decimal distance is shown in blue (or whatever colors are set in the color scheme of the active window).

 

The next indicator shows the length of the file.  It is shown in decimal if decimal addresses are being shown or in hex if addresses are currently shown in hex.  If the file has been edited and the length of the file has changed it also shows the difference in length in brackets.  When editing disks (raw disks or volumes) this shows the current sector and the total number of sectors.

 

The next indicator shows either LE or BE to indicate what endianness is currently being used (little-endian or big-endian).  Just double-click it to toggle it.  This setting is only important for operations that use or change values in the file -- certain Calculator buttons and commands on the Operations Menu.

 

The RO/RW indicator shows if the window is read-only -- if the file was opened read-only then this cannot be changed.  The OVR/INS indicator indicates if the window is in overtype or Insert mode.  It is common to work only in OVR mode as most types of binary files, such as executable files (.EXE, .DLL etc) database files etc, require that you do not insert or delete bytes.

 

The REC indicator shows when macro recording is on – double-click this indicator to start/stop recording.

 

 

Editing

 

HexEdit supports the typical ways of selecting bytes using the mouse or the shifted cursor keys plus cutting, copying, pasting, deleting etc. While you are selecting (either with the mouse or by holding down Shift and using the cursor keys) a popup tip displays the length of the current selection, either in hex or decimal depending on how addresses are currently displayed.  After the address is a number in brackets indicating divisibility by a binary power (1,2,4,8,...,256) – this is useful for some commands on the Operations Menu, which require the length of the selection to be divisible by 8, 4 or 2.

 

When you use the Cut command or Copy command the data is stored on the clipboard in two formats: text and binary. The text form allows pasting of text into an editor such as NotePad or a word processor. The binary form means that you can cut and paste in HexEdit without losing control characters (such as a byte with value zero); it also allows pasting binary data into the Visual Studio binary editor.

 

You can also copy to the clipboard as hex text or even as C/C++ source code, which is useful for initialising an array of bytes in your code. The Paste As ASCII, Paste As Unicode Text and Paste As EBCDIC Text commands allow you to perform such tasks as converting a block of text from EBCDIC to ASCII, removing invalid text bytes, or taking (ASCII) text from another program and pasting it as EBCDIC etc.

 

The Windows clipboard is limited to working with 16 Mbytes of data.  To get around this problem the Append Selection to File and Append Same File commands were added to allow you to easily create a new file composed of very large blocks selected from existing files.

 

As well as using the clipboard you can perform many other operations with the selected text such as writing the bytes to a file, highlighting, encrypting etc. A useful option available from the Print Dialog box is to print the selection (particularly as Windows does not allow printing of files with page numbers greater than about 32,000).

 

You can use the highlighter to mark particular important bytes within a file.  Clicking the highlighter, highlights the selected bytes.  To quickly create several highlights, turn on highlighter mode (click the highlighter button when there are no bytes selected), thence all subsequent selections made will be highlighted.  Highlighter mode can be used when comparing files to highlight all differences that were found.

 

Similar to the highlighter, but more powerful, are Bookmarks.  They simply allow you to associate a name with a location in a certain file.  (Of course, HexEdit keeps track of a bookmark's location if bytes are inserted or deleted before it).  Double-clicking a bookmark in the Bookmarks dialog allows you to jump straight to it, opening the file if necessary.  You can also quickly create unnamed bookmarks using Ctrl+F2 (in the default keyboard layout).  The Bookmarks dialog has other useful facilities such as the ability to sort the list of bookmarks by any column such as Name, time you created/modified the bookmark etc.

 

 

Undo

 

HexEdit has a two-level unlimited undo. Undo information is stored for cursor movement, changes to the display etc for each window. At the file level changes made to the file are also tracked. Changes to the file made in one window affect all windows open on that file. If you undo a change to the file all window level changes made in other windows of the file are undone back to when the file was changed – this is necessary to keep things consistent.

 

HexEdit does not store the bytes of the file in memory, it only keeps track of the changes made to the file (which are also used to undo the changes if necessary). This means you can easily edit very large files. But a consequence is that when you write a file to disk all the undo information for the file is deleted.

 

 

Properties

 

The Properties dialog (invoked with the Properties Command) not only allows you to view all sorts of values, but you can also change most values too. There are numerous options but in summary: you can edit the current byte as decimal, octal or binary; you can change decimal integer values (8/16/32/64 bit, signed/unsigned), floating point (IEEE/IBM, 32/64 bit), various date formats. Where appropriate, multibyte editing allows for big- or little-endian byte order.

 

The Properties Dialog is a “modeless” Dialog, so you can leave it open as you work.  This allows you to monitor values as you move the cursor around.

 

 

Tools

 

Quick searches can be performed with the Find Tool on the Edit Bar, although the (modeless) Find dialog provides more options. Unless wilcards are used searches are done with the Boyer-Moore algorithm which, apart from being very fast, has the peculiarity that the longer the string of (unique) characters you are searching for the faster the search, although the limiting factor is likely to be the speed of your disk or network.

 

When the background searches option is enabled (see System Options page), after you search for a new string a low-priority background thread is started to look for other occurrences of the same string in the same file (and other open files if any). This allows the occurrences to be highlighted and the next search for the same string is almost instantaneous.  The status bar search occurrences pane shows the number of occurrences present in the current file.

 

 

You can compare the data in any two (maximised) windows using Compare Windows command (usually mapped to Alt+C), even two parts of the same file. If a difference is found the offending byte is selected in both windows – pressing Alt+C again will continue the comparison at the following byte.

 

 

HexEdit has an integer calculator built into it. This allows many arithmetic, logical, and bit manipulation operations as well as base conversion. It is also useful for quickly calculating an address and jumping to that address using the “GO” button.  Of particular use are the calculator’s file access buttons, which allow you to manipulate values from the currently active file. These buttons mean the calculator is particularly useful when used in keystroke macros.

 

 

Macros

 

Keystroke macros allow you to repeat a repetitive task by recording it once and playing it back many times. Not only “keystrokes” are recorded but menu items, mouse selections and changing window focus etc. Setting the macro error level (see Macro Options page) allows you to stop runaway macros when something goes awry. But you can always abort a macro (or search, or comparison) by pressing the Escape key.

 

Macros can be saved to file and reloaded at a later time. If a macro called “autoexec” exists in the macros folder then it is run whenever HexEdit is started.  If a macro called "_xyz" (ie a file extesion preceded by an underscore) is created it is run whenever a file with the extension "xyz" is opened.

 

 

Templates

 

HexEdit templates offer a simple but extremely flexible way to describe the contents of certain types of files.  They not only allow for simple structures as in other hex editor templates, but also for repeated elements (with number of elements determined by count expression or termination condition), optional elements selected by a boolean (IF or IF-ELSE) expression or integer expression (CASE).  Templates derive their power from the ability to evaluate C-like expressions based on the data fields already seen in a file.  There is much more to templates than this such as the ability to create recursive data structures.

 

See Using Templates, Editing templates, Expressions, C/C++ Parser for Templates and Template Files for more details.  Also many have found these tutorials useful: Basic Template Tutorial, Advanced Template Tutorial.

 

 

Customization

 

There are many facilities for customization mainly available through the Customize command. You can customize the main menu and several context menus. For example, there are separate context menus when you right-click on the selection, highlighted bytes, bookmarks, the hex or character areas of the edit windows and even the address area, as well as the status bar.

 

You can create and completely customize the toolbars. You can even draw your own toolbar buttons with the built-in button editor. Similarly you can completely reconfigure the behaviour of the keyboard. You can even change the behaviour of mouse double-click events in different areas of the screen. Eg., by default a double-click in the address area selects the current line, but you could change it to toggle between the display of decimal and hex addresses.

 

The user tools customization allows you to set up menu items on the Tools menu to fire up external programs. For example, you can set up a user tool to fire up an editor (such as Notepad) to edit the current file.

 

 

Useful Keys

 

Most facilities are available from menus or toolbars. Often there are keyboard shortcuts for these which are displayed in the menu or in toolbar button tooltips (the hint that appears if you leave the mouse over a button for a second).

 

A few facilities are only accessible through keyboard shortcuts not associated with any menu or toolbar. You can find these in the Customize dialog under the “Non-menu commands” category or in the Keyboard Map command on the Help menu. Some of the more useful ones are shown below (along with their default keyboard shortcut):

 

Ctrl+L

Centre cursor within the window, and redraw the window

Ctrl+U

Scroll the window up

Ctrl+D

Scroll the window down

Tab

Swap between hex and character areas

Alt+Enter

Change the offset so the current column is at the left of display

Ctrl+F3

Search forward for next occurrence of currently selected bytes

 

You can abort a search, comparison or macro at any time by pressing the any key - you will be prompted if you want to abort.
