What’s New in 4.0

 

This version of HexEdit has been renamed to HexEdit Pro 4.0.  It contains a new, improved user interface and many other significant features.

User Interface

The major change to the user interface is the use of the new docking system first seen in Visual Studio but now common in other applications.  HexEdit Pro has always had several dockable windows (Bookmarks etc) that have always been able to be docked to the side of the window, moved to another part of the display such as a second monitor, or just "float" on top of the main HexEdit window.  These were previously called modeless dialogs, and dialog bars but are now called Tool Windows.

 

Tool Windows are still dockable and floatable, but can now be docked on top of each other, whence tabs appear to select between them.  Moreover, a tool window can be docked to the side of another tool window, and docking positions are much more flexible with visible indicators of the new position.  They can also be auto-hide.

 

However, many of the nice features of the old system have been moved to the new system such as the ability of floating tool windows to move out of the way when covering the current cursor position.  There is an option to turn this off if you don't like it.

 

There were also many other user interface improvements such as the Properties window being resizeable and being able to select the overall look of the application from the Application Look sub-menu of the View menu.

 

Further all simple dialogs that display a message or ask a simple question have been replaced with Windows 7 style information dialogs.  Many of these dialogs also have a checkbox to say "Do not tell me again" or "Do not ask me again".  There is a new option to reset all these settings so the message/question boxes are seen once more..

 

Other new features include

  • assign a keystroke macro to a keyboard shortcut, toolbar button, or menu item

  • move a bookmark simply by clicking and dragging

  • current byte used by the popup "tip" window has a rectangle around it in the hex and character areas

Code Pages

In addition to the existing character display options (ASCII, EBCDIC, etc) you can also select to display according to any of the installed Windows code pages.  This includes UTF-8, and many EBCDIC, Macintosh etc code pages.

Calculator

Extensive improvements have been made to the Calculator including:.

  • improved resizing behavior

  • support for use of any integer radix (not just 1,8,10,16) up to 36

  • support for signed/unsigned numbers in any radix

  • support for any size of integer arithmetic (not just 8, 16, 32 and 64)

  • arithmetic with infinite precision integers (signed or unsigned)

  • visual display of the bottom 64 bits of the current integer [below value display] - click any bit to toggle it

  • the full expression that is being calculated is displayed [above value display] including brackets to show order of evaluation

  • all calculations that have been performed are available from the drop-down history

  • history list also shows the calculated result for any item in a "tip" window

Installer

A new installer is now used which gives more reliable installations and new features.  When uninstalling you have the option of keeping or removing all settings.

Commands

Several new built-in CRC and message digest (SHA) commands have been added.  There is also general purpose CRC generator that allows generation of CRC according to several parameters (truncated polynomial etc) - not that this is a beta feature as it is not yet integrated with keystroke macros or documented in the help.

 

Other new commands are "Close Others" and several that change the application look,

Templates

One interesting features that was recently requested was the ability to assign a color to a template field so that the corresponding bytes in the hex area are highlighted in the selected color.

 

The C/C++ parser was enhanced to be compatible with the latest C standard.  In particular, variadic macros are now supported.

 

There were also several enhancements to HexEdit Pro's built-in expressions (see below).  (Expressions can be used in the calculator, jump tools etc but are mainly used in templates.)

Expressions

The sizeof(), defined() and string() functions now work on built-in variables.  (Previously they only worked on template fields.)

 

Date expressions have been improved in several ways.  First date differences return a value of FLOAT type rather than DATE type - this avoids some ambiguous situations caused by dates and date-differences both being stored in variables of DATE type.  (We could have invented a new "date difference" type but a FLOAT that simply stores the number of days between two dates is just as effective.)

Options

There are many new options:

  • clipboard: 4 new options for how binary data is copied and pasted (including the often requested "hex text")

  • calculate statistics (byte counts, CRCs etc) in a low-priority background thread (also see Properties below)

  • new options to not do background processing (search and statistics) for files on the network or on removable media

  • print a watermark behind all pages (including same options as header/footer for date etc)

  • different first page options for both headers and footers

  • six new options to turn on/off printing of bookmarks, highlights etc

  • swap left/right headers and footers on alternating pages (usually used with double-sided printing)

  • several options for the default open and save-as directories

  • option to occasionally check for the availability of updates

Properties

The Properties dialog is now resizable with many fields expanding to fill the space when it is enlarged.

 

Two new pages have been added to support the new background statistics options.  One page shows a graph of the byte count for all 256 byte values using the current color scheme.  Another page displays background values selected to be calculated such as CRCs of the file.
