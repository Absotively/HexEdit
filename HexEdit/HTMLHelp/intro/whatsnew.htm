What’s New in 3.6

 

Version 3.6 of HexEdit is an interim version before the release of HexEditPro 4.0.  Despite this there are several new features.

 

Installer

 

A new installer is now used which gives more reliable installations and new features.  When uninstalling you have the option of keeping or removing all settings.

 

User Interface

 

New commands have been added such as the Close Others command, which closes all other files except the active one.

 

New CRC and SHA1 commands were added.

 

New options have been added such as the Check for Updates option which checks for the availability of a new version of Hex Edit.  The check occurs no more than once a month to avoid being annoying.

 

The current byte for the Info-Tip window is highlighted when the tip window is shown.  Info-tips allow you to create custom tip windows which are shown when the mouse hovers over a byte but some people mentioned that they sometimes forget which byte they had been hovering over when the tip window appeared, especially as you can now drag the tip window where you want after it pops up.

 

Templates

 

Many fixes to the provided templates were added, some provided by users.  The PE (EXE/DLL) template has been fixed and enhanced for .Net meta-data.

 

The C/C++ parser was enhanced to be compatible with the latest C standard.  In particular, variadic macros are now supported.

 

There were also several enhancements to HexEdit's built-in expressions (see below).  (Expressions can be used in the calculator, jump tools etc but are mainly used in templates.)

 

Expressions

 

The sizeof(), defined() and string() functions now work on built-in variables.  (Previously they only worked on template fields.)

 

Date expressions have been improved in several ways.  First date differences return a value of FLOAT type rather than DATE type - this avoids some ambiguous situations caused by dates and date-differences both being stored in variables of DATE type.  (We could have invented a new "date difference" type but a FLOAT that simply stores the number of days between two dates is just as effective.)

 

Fixes

 

The "Save All" command prompted to save each modified files, which kind of defeated the purpose.  It has been fixed to silently save all modified files.

 

The recent files dialog was slow to open the first time.  This has been fixed.

 

Some special cases for the Decimal type (.Net) are now handled.  Bytes in this format can be viewed/edited in the Properties dialog.

 

The status bar pane sizing behaved strangely such that sometimes you could have a large message area (left pane) but panes on the right were not shown.  The message area pane now has a fixed minimum width, large enough to display most messages and only expands after all the other panes are visible.
