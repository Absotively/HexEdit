Technical Details

 

Bugs

 

For large files the print preview does not display the correct page numbers, and will only display up to the first 32767 pages.  A workaround is to select the part of the file to be printed and use the "selection" option in the Print dialog.

 

At the time of release there are no other known bugs.  For the latest bug information go to the Bug Forum on our Website.

 

Limitations

 

Limitations on the size of files that could be edited were removed with HexEdit 2.2.  The largest file you can edit is now much bigger than any existing disk file on the planet.  Unfortunately, the FAT filesystem limits files to 2 GBytes while FAT32 limits them to 4 GBytes.  However, if you use NT/2000/XP then you can edit files of virtually any size (NTFS has a limit on file size of one Terabyte).  The largest file edited so far is over 500 GBytes.

 

The maximum file size in HexEdit 2.1 and earlier is 2^31-1 bytes.  There was also previously a display limitation of 2^31 pixels.  This has been increased to 2^63 pixels in the vertical direction (but remains at 2^31 horizontally).  Using a font 16 pixels high (as the default Hex Edit font is) and a row size of 16 bytes this gives a vertical display limit of about 8 million Terabytes.  However, limitations in scrollbar calculations mean that scrollbars will not work correctly for files of more than about 500 Terabytes, I believe (but I have not tested this:-).

 

There is no limit on the number of undo operations or the number of “keys” in a keystroke macro except for the limitation of available virtual memory. Memory size would not normally be a problem as storage for these is very efficient. Storage for most undoable operations is 9 bytes; most macro “keystrokes” also use 9 bytes.

 

The maximum line length (ie, number of columns) is 32767.

 

There seems to be a limit of about 16Mbytes of data that can be copied to the clipboard in Windows 95 and 98. HexEdit will not let you put anything larger on the clipboard as it may cause HexEdit or even Windows to crash.  The workaround is to use the new commands for writing and appending to a file.

 

If a file needs more than 32767 pages to be printed in full then there is a problem with entering page numbers in the print dialog. This is a limitation of Microsoft Windows/MFC. To print parts of such a file use the “print selection” option.

 

Under Windows 95 files there is no standard place to store user specific files.  So files that are created for each user (bookmarks,recent file list etc) under later versions of Windows are stored in the executable directory under Windows 95.  In other words different logins on the same computer will share bookmarks, recent file list etc when running HexEdit under Windows 95.

 

Some early versions of Windows 95 did not support the Crypto API.  In this case the only encryption algorithm available is "Hex Edit Internal Encryption Algorithm".

 

International Support

 

Unfortunately HexEdit only currently supports English in its menus, messages, help etc. The documentation is in Australian English, which is very similar to English English and not entirely dissimilar to American English.

 

Luckily HexEdit does support international numeric and date formats when these are correctly configured in the Windows control panel.

 

Development Environment

 

HexEdit 1.0 was developed using Microsoft Visual C++ 5.0 and MFC. The system was NT 4.0 SR3 on a Pentium 100 system with 80 Mbytes of RAM and two hard disks (EIDE and SCSI).

 

HexEdit versions 1.1 to 2.2 were developed using Microsoft Visual C++ 6.0 under Windows 98 on a Celeron at 487.5 MHz with 128 Mbytes RAM and two monitors (19” and 15”).

 

HexEdit 2.5 to 3.1 were developed using VC++ 6.0, Windows XP on an Athlon 1600 (K7S5A mobo) with 512 Mbytes RAM and 2 monitors (19" @1600x1200 GForce 4) (17" @1024x768 ET6000).

 

Since version 3.2 we have been using Visual Studio 2003 (MFC 7.1) on XP SP2 with Sempron 2300, 1.5 Gbytes RAM.  (This is dual boot with Vista Ultimate but I never use Vista except for testing HexEdit on it.)
