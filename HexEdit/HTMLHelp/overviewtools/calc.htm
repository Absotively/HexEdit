Calculator

 

 

HexEdit includes a built-in calculator similar to the Windows calculator but with some enhancements and integration with the rest of HexEdit that make it very powerful. However, unlike the Windows calculator it only supports integer operations (although it does support 64 bit integers), and it does not use operator precedence or brackets.

 

The layout of the buttons is reminiscent of the Windows calculator but a little better organised. All binary operators are to the right of the numeric buttons, with unary operators to the left, and file access buttons to the far left.

 

Binary Operators

 

To perform a binary operation, enter the first operand (value) and click the desired operator button. You can still click a different operator if you accidentally clicked the wrong one. A small indicator appears to the right of the numeric entry box that indicates the current operator in use. After you enter the second operand you can then click the result button (=) or another binary operator to generate the result. If the operation generates an overflow or error then the calculator beeps and an E or O is displayed next to the numeric entry box.

 

As well as the normal arithmetic operators, and logical operators there are a few more esoteric operators. The LSL (logical shift left) and LSR (logical shift right) operators simply shift all the bits in the first operand (left or right) by the number in the second. The new bits added to the end always have a zero value.

 

The ASR (arithmetic shift right) operator, on the other hand, will add new bits with the value one if the current top bit is one. Whereas LSR is equivalent to a division by a power of 2 for an unsigned number, ASR is equivalent to a division by a power of 2 for a signed (2’s complement) number.

 

ROL (rotate left) and ROR (rotate right) are similar to LSL and LSR but the new bit shifted in is the bit shifted off the other end.

 

The GTR and LESS buttons find the larger or smaller of their two operands. These are useful in keystroke macros, particularly as the macro error level (see below) is set to a warning if the first operand is not greater (for GTR) or not less (for LESS) than the second.

 

The division operator (/) gives the truncated integer division, whereas the MOD operator gives the remainder. These are equivalent to the C/C++ integer / and % operators.

 

Unary Operators

 

Unary operators take a single operand -- the currently displayed number. Any current binary operator is not affected by a unary operation. All unary operator buttons are to the left of the numeric buttons except the change sign operator (+/-) that is between the 0 and = buttons.

 

There are arithmetic operators for square, factorial etc. Note that the square root operator truncates the result to the integer less than the real result. These operators can generate error and overflow conditions, which causes an E or O to be displayed next to the numeric entry box.

 

The ROL1, ROR1, LSL1, LSR1, and ASR1 buttons perform the same operation as their binary counterparts but shift by one bit only.

 

The INC and DEC buttons increment and decrement their operand. If this results in an overflow for the current operand size (see the discussion of Bits below) then a macro error is generated.

 

The REV button just reverses the order of all the bits in the current number. So if you are operating on 32 bit values bit 31 is swapped with bit 0, bit 30 with bit 1 etc. The FLIP button flips all the bytes in the current number. If you are working with 8 bit values this button is disabled since there is only one byte.

 

The @n operator takes the current number and uses it as an address into the currently active file. Using that address it reads the (8, 16, 32 or 64 bit) value from the file at that address. It is a unary operator as it takes an address and replaces it with the value at that address. See the File Access section below for more information.

 

Base and Bits

 

At the top of the calculator (below the numeric entry box) are two sets of radio buttons that allow you to choose the size of the numbers you are dealing with and in what base (radix) they are displayed. If you display 64 bit numbers in binary you will see up to 64 binary digits (0 or 1), which takes up the whole display.

 

The Base buttons allow you to enter numbers and display results in hexadecimal, decimal, octal or binary, and convert between those bases. Note that in decimal mode the numbers are considered to be signed, whereas in the other bases they are treated as unsigned, which may give slightly different results for negative numbers and different overflow conditions.

 

The Bits buttons allow you to specify the size of the numbers you want to work with. Note that if you change to a smaller number of bits and the current value won’t fit this will generate an overflow. For example, if the current value is 257 and you click on the 8 bit button an overflow is indicated and the current value changes to 1 (the value in the low byte of 257). To restore the original value change the number of bits back (eg. click on the 32 bit button).

 

Also note that many operations are done using the full 64 bits but only the requested number of bits are displayed. For example, if you add 255 and 2 in 8 bit mode an overflow is generated and the displayed result is 1. If you now click on the 32 bit button you can see the actual value of 257.

 

Furthermore, some operators (such as the logical operators AND, NOT etc) where the values of lower bits do not affect the value of higher bits always operate in 64 bit mode but only the requested number of bits in the result are displayed.

 

File Access

 

The buttons to the far left (except for the memory buttons) are only enabled when there is a file window active. They allow you to get the value of and even change the cursor, mark etc in the currently active file. Further, you can read and write values in the file.

 

The farthest column of buttons (Memory, Mark, @Mark, etc) are used to get values, whereas the buttons immediately to their right are used to change the corresponding value using the currently displayed number. There is also a unary operator button (@n) that can be used to get a value from the file – it is discussed above under Unary Operators.

 

The GO button (in the bottom right corner) is similar to the Set Cursor button but is also puts you back into file editing. This is the “default” button, so that if you type a number and press the Enter key, you will jump straight to the address you typed and can continue editing.

Note that there is no “Set EOF” button as you can’t change the length of the file in the calculator. (You can only change the length of the file if you are in INS (insert) mode and you add or delete bytes at the end of the file.)

 

The size of the integer read by the “@” buttons (@Mark, @Cursor, and @n) and written by the Store buttons is determined by the current Bits setting. The byte order of the numbers read from or written to the file is set by the Big-endian check-box near the top right corner of the calculator. (This is normally not checked allowing for Intel byte order.)

 

If you are not editing any files, the file access buttons are disabled (light grey), because there is no file to access. Some of the buttons may be disabled at various times for various reasons. For example, if the current displayed value is greater than the length of file then MkS, @n etc, that use that value as a file address, are disabled.

 

The file access buttons are particularly useful when used with keystroke macros (see previous section). For example, you could repeat the same calculation on every value in a file by storing the following operations in a keystroke macro: first load the value at the cursor using the @Cursor button, perform some operations on it, then save it back to the file using the Store button. To move to the next value in the file: load the current cursor address using the Cursor button, increment the required number of times depending on your operand size (using the INC button) and move the cursor using the Set button. Use the Tools/Macro Play menu item to replay the macro the required number of times (use a very big number to do the whole file).

 

Memory

 

To store the displayed number in the calculator memory, click the MS button. After a number has been stored, the Memory button is enabled, which you can later click to get the number back. The MC button clears the memory (and disables the Memory button). Note that you can generate an overflow just by recalling a number from the memory that is bigger than will fit in the current operand size.

 

The M+ and M- buttons allow you to add to and subtract from the current value in the memory.

 

The buttons below the memory buttons for the “mark” are analogous, but instead of changing the calculator memory they change the position of the mark in the active file window. They can be used as a makeshift second memory.

 

Shortcut Keys

 

The calculator provides many shortcut keys that are the same as those for the Windows calculator. For example, to recall the value of the calculator memory: press Ctrl+R. The shortcut keys are displayed in the “tips” that appear when you leave the mouse over a button for a second. The most common ones are for arithmetic operators: +, -, * (multiply), / (divide), and % (modulus); logical operators: &, | (OR), ^ (Exclusive-OR), and ~ (NOT); and = (Calculate).

 

Like the Windows calculator you can enter numbers by typing valid digits for the current base. The number displayed has spaces added for readability, but you don’t need to type these. Decimal numbers have commas as thousands separators (or whatever is set in the Control Panel for your culture/region).

 

Unlike the Windows calculator you can use the Home, End, Del, arrow keys etc to edit the current number. You can even select parts of the value using the Shift-arrow keys before deleting, cutting (Ctrl+X), copying (Ctrl+C) etc, and use Ctrl+V to paste from the clipboard.

 

Particularly useful are the keys to change the current base and size: F5 to F8 change betweens the four bases; F9-F12 change between the four sizes (64, 32, 16 or 8 bit).
