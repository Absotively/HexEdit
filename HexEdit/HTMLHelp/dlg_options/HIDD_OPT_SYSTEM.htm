System Options Page

 

 

The System options page allows you to change how HexEdit behaves globally.

 

Add “Open with HexEdit” to Shell shortcut menus

This allows any file to be opened with HexEdit using the shell shortcut menus which you see when you right-click on a file in Windows Explorer. The menu item says “Open with HexEdit”. When you click this menu item the file is opened in HexEdit.

 

Note that this option causes problems for some ill-behaved program launchers such as the Office 97 shortcut bar. If you use these then you will have to disable this option in HexEdit.

Only permit one instance of HexEdit to run

If you run HexEdit but HexEdit is already running then the new instance does not run, rather the original instance is brought to the front. Any files specified on the command line or via the shell are added to the files already open in the original instance.

Allow background searching

When the current search text is changed (either in the Find Tool or the Find dialog) a background search is started for all other occurrences of the same text. The search is done using low-priority background thread(s), so that you should not notice any difference to your normal use of HexEdit. (The search is done using a background thread with PRIORITY_LOWEST in the active file and even lower-priority (PRIORITY_IDLE) threads for any other open files.)

 

When the background search is finished the found occurrences are highlighted with a light-green background. Also when the background search is finished subsequent searches for the same search text are much faster – this is particularly noticeable for a large file.

 

You would normally leave this option on unless you had reason no to. You might turn it off if you do not need the facility and you are working with large files on a network and the file access is slowing down the network. Or perhaps you have another background task that is CPU intensive, so you want to give CPU time to that instead of the background searches.

Save settings on exit

Selecting this option forces all the current global settings to be saved when HexEdit is closed. Other less obvious things are also saved like the most recent positions of toolbars and modeless dialogs. A simpler way to customize your options is to set them how you want them and then use the Save settings now button (see below).

 

The storage for these options is in the file “HexEdit.ini” in the main Windows directory (usually C:\WINDOWS). A few options stored there cannot be changed from within HexEdit such as the default number of columns and column grouping in a window – you might like to modify these with a text editor.

Clear History Lists

Invokes the Clear Histories Dialog to delete search histories, bookmarks and the recently open file list, and have these automatically deleted each time you run HexEdit.

Recent File Entries

This sets the number of recently opened file entries displayed in the File menu.  You can enter any value between 1 and 16 inclusive.  If you want to open a file the you opened in the past but less recently than the last entry in the list then you can used the Recent Files Dialog.

Save settings now

Clicking this button forces all the current options to be saved immediately (even if the Save settings on exit option is not selected).

 

Create backup files

If this option is selected (and all of the conditions below are met) then whenever you save a file the previous version of the file, if any, is retained with the file extension of “.BAK”. (If you are editing a file with extension of “.BAK” the backup file has an extension of “.BAC”). The previous backup file, if any, is discarded.

 

To always create backup files turn on this option and turn off all the sub-options below.

Only if sufficient disk space

A backup is only created if there is free disk space (on the same volume) to accommodate the original file (which becomes the backup file) and the new file.  Note that this does not guarantee that the file will be able to be written to disk, since other processes or even the operating system may change the available disk space while the file is being written to disk.

Only if size is less than

Selecting this option allows you to specify a maximum file size.  If the (new) file is bigger than this a backup will not be created.

Prompt

Asks when the file is saved if you want to create a backup copy.  You are only asked if all the above options say that a backup should be created.

 

Use file selection address

When exporting the selection to a Motorola S or Intel Hex file you need to specify a starting address.  Selecting this option says to use the address within the file of the start of the selection.  For example, if you select and export the whole file the starting address will be zero..

Use address

When exporting the selection to a Motorola S or Intel Hex file this option allows you to specify the starting address, ie the address of the first line written to the file.  In this case the file address of the selection to be exported is ignored.

Bytes per line

This is the number of data bytes that are written to the exported text files created by the Export Motorola S1 Records, Export Motorola S2 Records, Export Motorola S3 Records, Export Intel Hex Records, and Export Hex Text commands.  The number should be kept within a reasonable range (and always between 8 and 250) as some software may have trouble reading very long lines.  A reasonable number would be 16 or 32.
