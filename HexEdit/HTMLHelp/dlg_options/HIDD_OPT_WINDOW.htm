Window Display Options Page

 

 

The Window Display page allows you to change options which only affect the display in the active window. (If there is no active window then this page is not available.)  You can also set the default settings for new files or files that have not been opened before.

 

Global Display Options

This button jumps to the page for more display options.

 

Show Area(s)

This list controls the main hex and character display in the active window.  You can display the Hex area, which allows you to view or edit the bytes in hex.  You can display the Character area, which allows you to see or edit the characters in the current character set in use (see Character Set below).  Or you can view both the hex and character areas together (Both Areas).  HexEdit 3.0 has added a fourth option which is called Stacked mode, where each byte is displayed as 3 bytes stacked vertically with the character at the top and the hex digits beneath.

 

Note that there is no way to turn off the address area on the left side of the display.

Columns

This field is the number of display columns in the active window. This field is disabled if the Adjust number of columns to fit the display (Autofit) option is selected.

Grouping

This field changes the grouping of columns in the hex display area. This also sets the number of columns by which the Ctrl+Left Arrow and Ctrl+Right Arrow keys move. The default value is 4 columns.

Offset

This field is the column number of the first byte of the file. The value must be between zero and one less than the Number of columns field.

 

This field is normally left at zero but it is sometimes useful to shift the columns around so that patterns in the data are more obvious. The starting offset is also changed by the Left Side command.

Autofit

This is the same as the Autofit command. When selected the number of columns is adjusted (whenever the window is resized) to fit the width of the window.

Maximize

This says whether the window is maximized.  This is the same as hitting the maximize button at the right end of the window's title bar or using the Maximize command on the window control menu.

Sector bounds

Says whether the sector boundaries are displayed.  The sector size depends on the disk but is often 512 bytes.  The color for displaying sectors can be set for the current color scheme in the Color Options page.

 

Hex Address

Display hex addresses on the left.

Decimal Address

Display decimal addresses in the address area on the left.  Also see Decimal Addresses command.

Line numbers

Display line numbers in the address area on the left..

Addresses start at 1

This allows addresses in the address area and offsets (in the ruler) to be shown starting from one (1) instead of zero (0).  By convention and to facilitate address calculations they usually start at zero, but some people prefer this option on.

 

Note that this only affects the display of addresses and line numbers in the address area and offsets in the ruler.  Other address display and calculation strictly uses zero-based addresses

 

Character Set

This list controls which character set is used for the display of characters.  If only the hex area is displayed (see Show Area(s) above) then this option is disabled.

 

The character sets available include EBCDIC and three ASCII variations.  The ASCII selection just displays a dot for characters with the high bit on (ie > 127) as ASCII is a 7-bit characters set.  The ASCII+ANSI set displays the ANSI extensions to ASCII which are used by most Windows fonts.  The ASCII+IBM/OEM option displays ASCII plus the graphics characters that were used in the first IBM PCs and are often associated with MSDOS games.  EBCDIC is a character set that IBM used in its mainframe and mini-computers.

 

Note that display of IBM/OEM graphics characters requires use of different Windows fonts.  Normally HexEdit uses the more common Windows ANSI fonts.  If you select the ASCII+IBM/OEM option you will see a different set of fonts when you select the current font (see the Font button above).

 

Also note that you can convert between different character sets using the Convert ASCII to EBCDIC, the Convert EBCDIC to ASCII, the Convert ANSI to IBM/OEM, and the Convert IBM/OEM to ANSI commands.

Control Characters

This list determines how control characters are displayed.  This option is disabled if only the hex area is hown or fif the character set is EBCDIC (since it does not use ASCII control characters) or IBM/OEM (since there are graphical characters in the ASCII control character range which are shown instead).

 

When None is chosen control characters are displayed as a dot.  When Alpha is chosen then the corresponding alphabetic character is displayed -- for example character 1 (Ctrl+A) is displayed as "a" in red (depending on the color scheme in use).  When C/C++ is chosen the C escape sequence for the control character is displayed - for example, character 9 (Tab) is displayed as "t" since at tab is represented as '\t' in C and C++.  Control characters without an escape sequence are displayed as a dot except for character zero which is displayed as a red "0".

Font

Click the Font button to select the font used in the active window.  This brings up the same Choose Font Dialog as the Font command.  Note that if you select the Display ASCII+IBM/OEM character set from the Character Set list (see below) then you will see a completely different set of fonts.  You may like to set both default fonts (the normal one and the IBM/OEM one).
