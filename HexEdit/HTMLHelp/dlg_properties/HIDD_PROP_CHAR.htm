Character Properties Page

 

 

This page shows information about the byte at the current cursor position of the active window. It also allows you to change the value as an unsigned decimal, octal or binary value. Note that all values are for the current byte at the cursor except for the Unicode field which uses two bytes starting at the cursor position. The Unicode field is disabled under Windows 95/98 as these systems cannot display Unicode characters.

Hex

The hexadecimal value of the byte from 00 to FF.

Decimal

Displays the unsigned decimal value of the byte from 0 to 255. To change it type a new value and press Enter (or press Escape to abort changes). Numbers above 255 are truncated. Negative values can be entered in the Byte field of the Decimal Values page.

Octal

Displays the octal value of the byte from 000 to 377. To change it type a new value and press Enter (or press Escape to abort changes).

Binary

Displays the binary value from 00000000 to 11111111. This field is mainly useful for examining the individual bits of the byte. “0” means the bit is off, “1” means it’s on. To change it type a new value and press Enter (or press Escape to abort changes).  Double-click any bit to flip it.

ASCII

The ASCII character at the current cursor position. If the high bit is set (ie, the decimal value is greater than 127) then this is actually the ANSI character. For values less than 32, this field displays the name of the ASCII control character.

Unicode

This is the Unicode character representing the two bytes following the cursor. This field is available under NT but disabled under Windows 95 (which does not support the display of Unicode characters).

Font

Use this button to select the font used for Unicode characters.  The font selected is remembered for the next time you run HexEdit..

EBCDIC

The EBCDIC character at the current cursor position. The EBCDIC control character name is displayed for values less than 64, and “none” is displayed for invalid EBCDIC characters.
